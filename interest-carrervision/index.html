now carrer vision

将来的には、ネットワークやサーバーを中心としたインフラ領域で、設計・構築・運用・トラブルシューティングを横断的に担えるエンジニアを目指しています。

CTF や ICTSC への参加を通じて、「設定を覚える」だけでなく、「なぜその設計・設定が必要なのか」を考え、障害発生時に冷静に切り分ける力の重要性を実感しました。

この経験を活かし、実際の運用現場に近い視点を持ったエンジニアとして成長していきたいと考えています。

また、個人でのサーバー運用や Web サイト構築を通じて、ネットワークとサービスがどのように結びつき、ユーザーに価値を届けているのかを意識するようになりました。

今後は、大規模ネットワーク環境や、セキュリティを考慮したインフラ設計・運用に携わり、現場で信頼されるエンジニアになることを目標としています。

interest

漠然とセキュリティ分野に興味を持ち、ホワイトハッカーに憧れて進学先学科を選びました。

大学入学後、ネットワークやサーバーを実機・仮想環境で触れる機会を通して、インフラ分野そのものの面白さに強く惹かれるようになりました。

特に、ログや設定ファイルを辿りながら原因を切り分け、通信が成立するまでの過程に大きな魅力を感じています。

GUI よりも CUI での操作を好み、ターミナル上で試行錯誤を重ねる時間に強い没頭感と達成感を覚えます。

Cisco Packet Tracer を用いたルーティング構築、Proxmox 上での仮想サーバー構築、WireGuard による VPN、権威 DNS サーバー運用などを通して、「ネットワークがなぜ動くのか」「どこで問題が起きるのか」を自分の手で確かめることに関心を持つようになりました。

現在は、OSPF・iBGP・BGP、IPv4 / IPv6 環境での疎通確認・経路監視など、より実運用に近いネットワーク構成や挙動の理解に強い興味を持っています。
