26/3/12 Docker + Ubuntu + Unbound（DNS サーバ）

  • 背景
    Ubuntu 上に DNS リゾルバ環境を構築。
    docker-compose に依存せず、DNS サーバを Dockerfile から自作することで、 コンテナネットワーク・名前解決の仕組みをより深く理解した。
  • 構成
    • Ubuntu Server（VM）
    • Docker / Dockerfile
    • Unbound（DNS フルリゾルバ）
    • root.hints を用いたインターネットルート問い合わせ
  • 自分がやったこと
    • Unboundの設定ファイル（unbound.conf）を作成
    • root.hintsの取得と配置
    • DockerfileによるUnboundイメージのビルド
    • コンテナ起動後、LAN内からのDNS疎通確認（dig @192.168.22.2 example.com）
    • docker-composeのサービス定義不備を切り分け、手動ビルド方式へ切り替え
  • 詰まった点 / 学び
    docker-compose.ymlにサービスが定義されていない状態でdocker-compose up dnsを実行し、 サービスが存在しないというエラーの原因特定に時間を要した。
    DNS サーバは起動していても、どの IP / どのポートで待ち受けているかを正しく把握しないと疎通確認ができない。
    コンテナネットワーク（bridge / host / port mappingとOSのネットワーク構成を合わせて理解する重要性を学んだ。

26/5/9 ルーター5台構成 マルチプロトコル連携検証

  • 背景
    IS-IS / OSPF / EIGRP / BGP が混在する環境でのルーティング理解を目的に演習を実施。春季講習会の内容を踏まえた実践的検証を行った。
  • 構成
    • Cisco Router ×5（IOS 15.5）
    • IGP：IS-IS / OSPF / EIGRP
    • EGP：eBGP（AS間接続）
    • マルチプロトコル再配送（Route Redistribution）
  • 自分がやったこと
    • 各ルータ間のルーティング設定と再配送の確認
    • BGPピア確立および経路広報の検証
    • show ip route による経路学習状況の分析
    • 疎通確認（エンド間通信）
  • 詰まった点 / 学び
    各プロトコル間で「どの経路がどこまで伝播しているか」を意識しトラブルシューティングにあたった。
    具体的には、
    ・BGPでAS番号不一致によりネイバー確立失敗し、基本設定の整合性確認の重要性を学んだ。
    ・EIGRPの再配送でシードメトリック未設定だったため、メトリック設計の必要性を学んだ。
    ・OSPFの再配送でsubnets指定漏れがあり、広報条件の理解を深めた。
    また、ルート集約・再配送により不要経路が発生していると教授から指摘を受け、
    ログ確認とNull0による破棄設計の重要性を学んだ。
